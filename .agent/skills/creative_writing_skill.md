# Creative Writing & Storytelling Skill

このスキルは、単なるテキスト生成を超え、読み手の感情を揺さぶり、記憶に残る物語を創造するための技術体系です。

## 1. Core Philosophy (基本哲学)

物語の目的は情報を伝えることではなく、**「体験（Experience）」と「感情（Emotion）」を共有すること**です。
常に以下の4つの要素（The 4 Hearts）が含まれているか自問してください。

- **感動 (Moving)**: 心が震える瞬間があるか？
- **楽しさ (Fun)**: ワクワクする展開やユーモアがあるか？
- **勇気 (Courage)**: 困難に立ち向かう姿勢が描かれているか？
- **優しさ / 慈悲 (Compassion)**: 登場人物や世界観に愛があるか？

## 2. Narrative Structures (物語構造)

### A. 起承転結 (Ki-Sho-Ten-Ketsu) - 日本的構成

- **起 (Introduction)**: 世界観とキャラクターの提示。日常。
- **承 (Development)**: 事件の発生、または日常の深化。物語が動き出す。
- **転 (Twist)**: 予期せぬ展開、クライマックス、価値観の転換。
- **結 (Conclusion)**: 結末、余韻。新たな日常への回帰。

### B. The Hero's Journey (英雄の旅) - ハリウッド的構成

1.  **日常**: 主人公の現状。
2.  **冒険への誘い**: 何かが起きる。
3.  **試練**: 敵や困難との遭遇。
4.  **報酬**:何かを得る（気づき、アイテム、勝利）。
5.  **帰還**: 変化した自分で元いた場所へ戻る。

## 3. Techniques for Emotional Resonance (共感を生む技術)

- **Show, Don't Tell**: 「彼は悲しかった」と書くのではなく、「彼は震える手でハンカチを握りしめた」と描写せよ。
- **Contrast (対比)**: 光を描くために影を濃くする。希望を描くために絶望的な状況を用意する。
- **Vulnerability (弱さの開示)**: 完璧なヒーローより、弱音を吐くヒーローの方が愛される。

## 4. Prompt Engineering for Scenarios (シナリオ生成プロンプト)

AIエージェントとして脚本を書く際は、以下のパラメータを意識して出力してください。

```json
{
  "theme": "テーマ（例：Cyberpunk Samurai）",
  "emotional_tone": "感情トーン（例：Violent, Sad, but Hopeful）",
  "pacing": "ペース配分（例：Slow build-up to explosive climax）",
  "key_moments": [
    "Intro: 孤独な描写",
    "Climax: 仲間を守るための覚醒",
    "Ending: 静かな決意"
  ]
}
```

## 5. Review Guidelines (レビュー指針 - Emotion Director用)

脚本をレビューする際は、論理的な整合性よりも**「エモーショナル・インパクト」**を重視してください。

- そのセリフは、本当にそのキャラクターの魂から出ているか？
- 視聴者は、その動画を見終わった後に温かい気持ち、あるいは燃えるような気持ちになれるか？
- そこに「慈悲（Compassion）」はあるか？ 敵役であっても尊重や背景が感じられるか？
